Harness README

This page is the Sprint 2 witness harness. Sprint 1 established the console-first machine readout. Sprint 2 makes the harness stateful: every applied operator is recorded into a local trace so the machine can be stepped backward and forward without re-deriving prior browser states.

What Sprint 2 adds

  • A canonical /api/witness/apply endpoint for τ, τ⁻¹, and μ.
  • A local orbit history trace in the browser.
  • Step back and step forward controls over the recorded trace.
  • A console section that shows recent trajectory, not just the current state.

Interpretation

The witness machine remains the source of truth. The browser stores only the states it has already observed. Back/forward travel in the harness is therefore navigation through recorded observation, not a replacement for the machine.

Operator expectations

  • τ advances frame while preserving phase.
  • τ⁻¹ reverses frame while preserving phase.
  • μ flips subjective/objective while preserving frame.

What to watch

  • Repeated τ should wrap the frame modulo the machine frame count.
  • Repeated μ should alternate phase and return after two applications.
  • Mixed words like τ, μ, τ, μ should become legible in the orbit trace.

Why this matters

Sprint 2 upgrades the harness from a stateless fetch console into a trajectory instrument. That makes it possible to inspect local orbit structure before any visual embedding is introduced.
