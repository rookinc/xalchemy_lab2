# G15 transport cocycle

This file is the anti-drift anchor for the transition law of the G15 transport shell.

## Core idea

The cocycle is not a separate object floating in space.

It is the lawful transition data that carries the walk description from one local description to another while preserving the underlying transport event.

## Where it lives

The cocycle lives on overlaps, especially:

- between downstairs and stairs
- between stairs and upstairs
- between one visual lens and another

## What it carries

A local walk description may include:

- anchor
- orientation
- region
- register slot
- phase
- closure state
- lens

The cocycle tells us how that description changes when we pass through an overlap.

## Three transition classes

### Region transition
Carries the walk state between:

- downstairs
- stairs
- upstairs

### Lens transition
Re-expresses the same transport event in another visual lens:

- sheet lens
- cube lens
- tetra lens

### Stairs transition
The special transition centered on the stairs, where level change and reorientation are most visible.

## Stairs principle

The stairs are the distinguished overlap region.

In cube lens:

- the walk is read as starting and stopping on the stairs
- v0 is the distinguished stairs midpoint / cube centroid anchor

## What the cocycle may change

The cocycle can change the local description by carrying:

- region change
- orientation change
- register shift
- lens relabeling
- closure transport

## What the cocycle does not change

The cocycle does **not** change the underlying transport event.

It does **not** change the theorem object.

It does **not** renumber the exact transport matrix.

## House picture

Downstairs and upstairs are different local floorplans.

The stairs landing is where the translation happens.

The cocycle is the rule that says how a walk state downstairs is re-expressed upstairs while remaining the same transport event.
