# d4 Prime Growth Spec v0.2

## Status
Working spec

## Purpose
Define the prime tetra growth law, composite layer semantics, centroid-based orientation, chirality transport, ordered mirror/cohesion expansion, and the currently derived working answers to the major open questions.

---

## 1. Core objects

### 1.1 Prime
A prime is the fundamental tetrahedral growth unit.

A prime has:

- 4 vertices
- 6 edges
- 4 geometric triangular faces
- 1 composite downward carrier layer
- 1 chirality state
- 1 centroidal orientation frame

### 1.2 Geometric faces
The geometric face labels are fixed:

- Face 1 = growth face
- Face 2 = growth face
- Face 3 = growth face
- Face 4 = composite downward carrier face

### 1.3 Composite layer
Face 4 is not treated only as a single terminal triangle.

Face 4 is the carrier of an expanded pleated composite layer indexed by:

- 4, 5, 6, 7, 8, 9, 10, 11, 12

Interpretation:

- 4 = composite carrier identity / base center
- 5..12 = pleat addresses on the composite layer

These pleat indices are semantic/control addresses, not additional Euclidean outer tetra faces.

---

## 2. Centroidal frame

### 2.1 Principle
All local orientation is defined from the centroidal frame.

### 2.2 Cell frame
Each prime has a local frame:

- origin = cell centroid
- down_axis = normal of face 4
- up_axis = opposite of down_axis
- north = centroid-derived tangent direction
- east = right-handed tangent complement

### 2.3 Composite frame
The composite layer has its own local frame:

- composite_origin = centroid of face 4
- composite_down = face 4 normal
- composite_north = canonical tangent direction anchored at composite centroid
- pleat ordering proceeds cyclically from north

### 2.4 Consequence
North is never screen-up by default.
North is always centroid-derived.

---

## 3. Chirality

### 3.1 States
Each prime carries a chirality state:

- left
- right

### 3.2 Complementarity
The initial tetrahedron is resolved from four composite faces with complementary chirality.

### 3.3 Mirror law
When a new prime is generated by mirroring through an active face, the child prime takes the complementary chirality of the parent.

So:

- left -> right
- right -> left

---

## 4. Seed rule

### 4.1 Pre-prime emergence
The seed tetrahedron t_0 is not treated as arbitrary input.
It is resolved from a 4-face composite closure.

### 4.2 Seed orientation
The seed tetrahedron must satisfy:

- face 4 is the downward composite carrier
- faces 1,2,3 are the growth faces
- centroidal frame is initialized
- chirality is assigned

---

## 5. Growth law

### 5.1 Primitive operation
The primitive growth operation is ordered face mirroring.

At each turn:

1. choose the active cohesion surface
2. mirror the active tetra through that face
3. generate exactly one child prime
4. flip chirality
5. update composite pleat state
6. update open/closed topology
7. advance turn index

### 5.2 One-turn rule
Each turn performs exactly one surface-cohesion event.

There is no unconstrained simultaneous branching in the primitive law.

### 5.3 Ordered face schedule
Primary face order is:

- 1
- then 2
- then 3
- then repeat

This is the local ordered face schedule.

---

## 6. 3:6:9 cycle

### 6.1 Cycle meaning
The ordered face schedule is grouped into a higher cycle:

- +3
- +6
- +9

### 6.2 Interpretation
Working interpretation:

- +3 = first ordered pass across primary growth faces
- +6 = secondary complementary pass
- +9 = composite/pleated consolidation pass

### 6.3 Cycle sum
One full cycle has sum:

- 3 + 6 + 9 = 18

### 6.4 Cycle-complete milestones
Counts that are multiples of 18 are cycle-complete checkpoints.

Example:

- 900 = 18 x 50

This is a lawful milestone, but not the only valid pause point.

## 7. Composite pleat semantics

### 7.1 Carrier
Face 4 remains the composite carrier at all times.

### 7.2 Pleat registry
The expanded composite layer is represented by the pleat registry:

- 4..12

Each pleat can carry state such as:

- open
- closed
- occupied
- cohered
- mirrored
- reserved

### 7.3 Purpose
Pleats are used for:

- composite-layer bookkeeping
- local orientation inheritance
- ordered cohesion routing
- chamber bookkeeping

## 8. Topological bookkeeping

### 8.1 Vertex state
Vertices are tracked as:

- open
- closed

### 8.2 Face state
Faces are tracked as:

- exposed
- internal
- composite
- pleated

### 8.3 Cell state
Each cell tracks:

- tetra id
- parent tetra
- generating face
- chirality
- centroidal frame
- composite pleat state

### 8.4 Chamber state
A chamber is a grouped closure structure built from lawful prime propagation.

Chambers are tracked independently from raw tetra count.

## 9. Viewer semantics

### 9.1 Pause control
The user control is not a sacred target.
It is a soft stop:

- pause_at_d4s

The engine grows from current count upward.
When current count reaches pause_at_d4s, playback auto-pauses.

Pressing play again allows growth beyond that point.
Reset returns to 0.

### 9.2 Display modes
Recommended display modes:

- Scaffold
- Prime
- Prime+Composite
- Hybrid

### 9.3 Prime display
Prime display renders actual adhered tetra cells.

### 9.4 Composite display
Composite display overlays the expanded pleated carrier state on face 4.

### 9.5 Hybrid display
Hybrid shows both scaffold flow and prime/composite structure in the same camera frame.

11. Engine requirements

The engine must:

1. preserve face labels 1,2,3,4


2. preserve face 4 as the composite downward carrier


3. derive all local orientation from the centroidal frame


4. flip chirality under mirror propagation


5. perform exactly one cohesion event per turn


6. honor local ordered face progression 1 -> 2 -> 3


7. track higher 3:6:9 cycle state


8. track open/closed vertices and exposed/internal faces


9. support arbitrary pause thresholds


10. preserve camera state across display mode switches

12. Derived working answers

12.1 Pleat naming and order

The current working pleat order is:

4 = carrier center

5 = north

6 = north-east

7 = east

8 = south-east

9 = south

10 = south-west

11 = west

12 = north-west


This order is defined cyclically from centroid-derived north in the composite frame.

12.2 Chamber definition

A chamber is a local closure packet formed when a prime and its ordered mirrored descendants complete a coherent primary surface round around a common carrier context.

Working operational approximation:

one chamber corresponds to one completed local 1 -> 2 -> 3 coherence round


12.3 Relation of pleats to the +9 phase

The +9 phase is the composite consolidation phase.

It corresponds to a full update sweep across the composite carrier registry:

4 = carrier center

5..12 = eight pleats


So +9 is interpreted as a complete composite-layer update over 4..12.

12.4 900

900 is definitely a lawful cycle-complete milestone because:

3 + 6 + 9 = 18

900 = 18 x 50


So 900 is at minimum a cycle-complete checkpoint.
